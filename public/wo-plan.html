03 — Flow utilisateur : sélection de lignes
👆 Clic checkbox sur une ligne
1
AG Grid déclenche selectionChanged
Le composable reçoit les nœuds via getSelectedNodes()
2
Vérification par isSelectable(wo)
Statut dans BULK_EDITABLE_STATUSES + résultat ≠ Negative
3
Désélection silencieuse des invalides
setNodesSelected({ newValue: false }) sur les nœuds rejetés
4
Store mis à jour
store.setSelection(validIds) — sans toucher au draft
5
Computed sharedStatus recalculé
Statuts hétérogènes → panneau masqué. Identiques → panneau visible.
🚫 Cas WO Result = Negative
1
Checkbox visiblement grisée
isRowSelectable retourne false → AG Grid désactive nativement
2
Ligne colorée (rouge atténué)
Via rowClassRules["wo-row--negative"]
3
Tooltip sur la checkbox
"Résultat négatif — statut non modifiable"
4
Exception : Report envoyable
buildBulkPayload inclut le report. Statut exclu du payload.

06 — Flow soumission & gestion des erreurs
Bouton "Appliquer" cliqué
Validation synchrone d'abord, puis appel API
① VALIDATION
canSubmit = false si :
• validationErrors.length > 0
• isSubmitting = true
• showBulkPanel = false
② BUILD PAYLOAD
service.buildBulkPayload()
• Toutes les lignes sélectionnées
• Exclut statut si Negative
• Inclut report pour tous
③ APPEL API
PATCH /bulk-edit/
• isSubmitting = true
• try/catch réseau
• Erreur → submitError global
④ RÉPONSE BACK
partitionBulkResults()
• succeeded → applyDraftToRows()
• failed → lastResults
• submitError = "N WO en erreur"
⑤ AFFICHAGE ERREURS
Pour chaque WO failed :
• WO #ID dans une liste
• Message d'erreur du back
• Draft conservé
• Sélection maintenue
